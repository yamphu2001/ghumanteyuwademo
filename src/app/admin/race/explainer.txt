matchmaking.ts — the core engine. Runs every 10s on the admin side, reads the /lobby from Firebase Realtime Database FIFO by requestTime, fills open groups (max 10 players, Option B bin-filling), closes full groups, and removes assigned players from the lobby atomically. Skips inactive players (isActive: false). Exports startMatchmaking(), stopMatchmaking(), and subscribeToGroups() for live x/10 UI updates.
test.ts — player simulator. Spawns fake Nepali-named players into the lobby every 2s to test without real clients. Has a 10% inactive player chance for edge case testing. Exports startSimulator(), stopSimulator(), spawnOne(), and clearAll() for full Firebase resets between runs.
page.tsx — the admin test panel UI. Dark themed, matching Ghumante Yuwa's #FF5F1F branding. Shows live group cards with pip indicators and progress bars, a next-tick countdown, stats row, engine controls, and an activity log.
Firebase side — added .indexOn: requestTime to the lobby rules so the FIFO query works efficiently.
